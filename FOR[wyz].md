# 仿真看板项目 - 学习笔记

## 项目背景：为什么做这个？

想象你在管理一个大型自动驾驶仿真平台，每天有7个团队在上面运行成百上千个仿真任务。作为平台管理者，你有个很朴素的需求：**想知道各团队用得怎么样**。

- 感知算法部这周发起了多少任务？
- 哪些团队活跃度高，哪些几乎不用？
- 使用人数的趋势是上升还是下降？

这些问题听起来简单，但当时平台居然没有现成的看板。要么去数据库里手写SQL查，要么只能靠感觉。所以我们决定：**做一个轻量级的数据监控看板**。

### 为什么选择纯HTML+JS方案？

当时有几个技术选项：

| 方案 | 优点 | 缺点 | 选择 |
|------|------|------|------|
| React + ECharts | 功能强大 | 需要构建、部署复杂 | ❌ 太重 |
| Vue + Element UI | 组件丰富 | 同样需要构建工具 | ❌ 过度设计 |
| **纯HTML+JS+SVG** | **单文件、即开即用** | 功能需要自己写 | ✅ **选中** |
| Excel/BI工具 | 快速出报表 | 不灵活、不够实时 | ❌ 不够酷 |

**决策逻辑：**
1. **时间紧迫** - 第一期要快速验证需求，不想折腾构建配置
2. **部署简单** - 单个HTML文件，扔哪都能跑
3. **学习价值** - 手写SVG图表是个很好的练手机会
4. **够用就好** - 不需要复杂的状态管理，就是展示数据

> 💡 **工程师思维：** 不要上来就用最先进的技术栈。问自己：**最简单的方式能解决问题吗？** 能的话就用简单的。

## 经验教训与踩坑记录

### 坑1：SVG坐标系统

**问题：** 一开始图表画出来是空的，怎么都显示不出来。

**原因：** SVG有自己的坐标系统，(0,0)在左上角，y轴向下增长。而我习惯了数学坐标系（y轴向上）。

**解决：**
```javascript
// 错误思维（数学坐标系）
const y = value / maxValue * height;

// 正确思维（SVG坐标系）
const y = padding.top + (height - padding.top - padding.bottom) * (1 - value / maxValue);
```

**经验：** 搞图形编程要先搞清楚坐标系统，不然算半天都是错的。

---

### 坑2：日期格式的坑

**问题：** 模拟数据生成后，图表显示不出来。

**原因：** 日期字符串比较出了问题。`'2026-3-1'` 和 `'2026-03-01'` 被当成不同的日期。

**解决：**
```javascript
// 统一日期格式，补零
function formatDate(date) {
  const year = date.getFullYear();
  const month = String(date.getMonth() + 1).padStart(2, '0');  // 关键
  const day = String(date.getDate()).padStart(2, '0');
  return `${year}-${month}-${day}`;
}
```

**经验：** 日期处理一定要统一格式，特别是月份和日期要补零。

---

### 坑3：悬浮提示定位

**问题：** 提示框位置不对，有时候跑到屏幕外面去了。

**原因：** 用的是`e.target.getBoundingClientRect()`获取位置，但没考虑tooltip自身宽度。

**解决：**
```javascript
// 简单居中定位
tooltip.style.left = rect.left + rect.width / 2 - 90 + 'px';  // 90是tooltip宽度的一半
tooltip.style.top = rect.top - 100 + 'px';
```

**更好的方案（未来改进）：** 动态检测边界，防止溢出。

---

### 最佳实践1：预留API接口

**做法：** 在CONFIG对象中预留了`useMockData`开关。

**好处：**
- 第一期用模拟数据快速开发
- 第二期只需要改配置，不需要重写代码
- 出问题时可以快速切换回模拟数据验证

```javascript
const CONFIG = {
  useMockData: true,  // 一期：true  二期：false
  apiBaseUrl: '...',
  apiToken: '...'
};
```

> 💡 **工程师思维：** 写代码时想想未来会怎么改。提前设计好扩展点，后面会省很多事。

---

### 最佳实践2：模拟数据要逼真

**做法：** 不是随机生成，而是加上趋势、周末效应、波动性。

```javascript
// 不是这样（太假）
const tasks = Math.random() * 100;

// 而是这样（像真的）
const trendFactor = 1 + (config.trend * index);  // 趋势
const weekendFactor = isWeekend ? 0.3 : 1;        // 周末少
const volatility = Math.random() * config.volatility;  // 随机波动
const tasks = baseTasks * trendFactor * weekendFactor * (1 + volatility);
```

**好处：** 模拟数据看着真实，能暴露真实场景的问题。
